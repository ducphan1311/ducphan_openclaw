## Entry Points

### Docker Mode

- `docker-compose.yml:4` defines `openclaw_agent`.
- `docker-compose.yml:7` runs `openclaw gateway --allow-unconfigured --port 18789`.
- `docker-compose.yml:23` mounts local `workspace/`, `skills/`, `config/`, and `openclaw_data/` into the container.
- `entrypoint.sh:4` fetches Vault secrets when `VAULT_TOKEN` is present.
- `entrypoint.sh:55` syncs approved environment variables into OpenClaw runtime config.

### Native macOS Mode

- `start_native.sh:10` sets workspace, skills, config, and data paths under this repo.
- `start_native.sh:16` sources `.env` when present.
- `start_native.sh:24` sets default local Vault address.
- `start_native.sh:88` exports OpenClaw runtime settings.
- `start_native.sh:103` installs `openclaw`, `playwright`, and `pnpm` globally if missing.
- `start_native.sh:119` writes local OpenClaw config into `openclaw_data/openclaw.json`.
- `start_native.sh:247` starts the gateway on port `18789`.

## Working Directories

- `workspace/` is for generated plans, memory, research notes, shopping watchlists, and travel notes.
- `skills/` is the workspace skill root loaded by OpenClaw.
- `config/` contains OpenClaw policy/config.
- `openclaw_data/` contains local runtime state and is intentionally ignored.
- `vault/` contains Vault config/state; secrets should stay in Vault storage, not source files.
